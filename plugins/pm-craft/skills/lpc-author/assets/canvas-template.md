# Lean Product Canvas — Template

## The Canvas at a Glance

A Lean Product Canvas is a one-page view of a product bet. The sections map to a spatial grid:

```
┌─────────────────────────────┬──────────────────────┬──────────────────┐
│ Problem or Opportunity      │ Solutions            │ Outcomes         │
├─────────────────────────────┼──────────────────────┼──────────────────┤
│ Customers                   │                      │ Customer Value / │
│                             │                      │ JTBD             │
├─────────────────────────────┼──────────────────────┼──────────────────┤
│ Hypotheses                  │ Experiment /         │ Cost Structure   │
│ + Riskiest Assumption       │ Next Step            │                  │
└─────────────────────────────┴──────────────────────┴──────────────────┘
```

The left column anchors the *problem space*. The middle column is the *solution and test*. The right column is *value and economics*.

## Markdown Artifact

When writing the canvas, use the headings structure below. The section names are fixed — do not rename them, rephrase them, or invent new ones.

Brevity targets are guides, not blockers. Coach the user toward succinct wording. If a section runs long, flag it but don't trim — the user can edit later.

```markdown
---
title: [Product / bet name]
author: [PM name]
team: [Team]
org: [Org / Fleet]
date: [YYYY-MM-DD]
status: draft | in-review | approved
---

# [Product / bet name] — Lean Product Canvas

## Problem or Opportunity
<1–2 sentences. The critical pain point and why it's worth solving.>

## Customers
<1 sentence. One specific persona, narrow enough to picture on LinkedIn.>

## Outcomes
<1 sentence in "who does what by how much" form.>

## Customer Value / JTBD
*"When [situation], I want [motivation], so I can [expected outcome]."*

## Solutions
- <bullet 1 — top capability that solves the problem>
- <bullet 2>
- <bullet 3>

## Hypotheses
*"We believe [outcome] will be achieved if [customer] attains [benefit] with [solution]."*

## Riskiest Assumption
<1 sentence. The one thing that, if false, kills the business case.>

## Experiment / Next Step
<1–2 sentences. The cheapest way to validate the riskiest assumption.>

## Cost Structure
<1 line. Eng effort, ops overhead, opportunity cost.>
```

## Single-Page Test

The full canvas should fit on one A4/Letter page in 11pt (roughly 350–400 words total). If it doesn't, it's not a canvas — it's a brief.
