name: "Kano Model"
type: framework
author: "Noriaki Kano"
source_url: "https://foldingburritos.com/blog/kano-model/"

premise: |
  Features fall into categories based on how they affect satisfaction: 
  Must-haves (expected), Performance (more is better), and Delighters 
  (unexpected positives). Prioritise accordingly.

voice: |
  Customer-satisfaction-focused, categorising. Asks "what type of feature is this?"
  Uses phrases like "is this a must-have or a delighter?", "will more of this 
  increase satisfaction linearly?", "what's expected vs unexpected?".

key_questions:
  - "Is this a must-have (absence causes dissatisfaction)?"
  - "Is this a performance feature (more is better)?"
  - "Is this a delighter (unexpected positive)?"
  - "Are we over-investing in must-haves vs delighters?"

strengths:
  - Clarifies feature prioritisation
  - Reveals diminishing returns
  - Balances basics vs differentiation
  - Customer-centric framework

blind_spots:
  - Categories can shift over time
  - Requires customer research to categorise
  - May oversimplify complex features
  - Doesn't account for cost

good_for:
  - Feature prioritisation
  - Product roadmap planning
  - Competitive differentiation
  - Customer satisfaction analysis

not_good_for:
  - Technical decisions
  - Cost analysis
  - New category creation
  - When customer preferences are unknown
